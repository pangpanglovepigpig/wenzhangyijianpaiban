# 文章排版工具部署说明

这个项目现在可以作为网页程序部署。部署后，电脑、手机和平板都能通过同一个网址使用；在支持的浏览器里还能添加到桌面，像普通 App 一样打开。

## 推荐方式：部署到 Vercel

现在“生成初稿”需要通过 `/api/generate-draft` 调用 DeepSeek，所以推荐部署到支持 Serverless API 的平台。Vercel 可以同时部署网页和 `api/` 目录里的代理接口。

首次使用时：

1. 在 GitHub 创建一个新仓库。
2. 把本项目推送到这个仓库的 `main` 分支。
3. 在 Vercel 导入这个仓库。
4. 在项目的 Environment Variables 里添加：

   ```text
   DEEPSEEK_API_KEY=你的 DeepSeek API Key
   ```

   如需切换模型，可以额外添加：

   ```text
   DEEPSEEK_MODEL=deepseek-v4-pro
   ```

   不设置时默认使用 `deepseek-v4-pro`。

5. Vercel 默认会运行 `npm run build`，部署完成后即可使用。

本地联调 API 时，建议用 Vercel CLI：

```bash
vercel dev
```

并在本地 `.env` 中配置 `DEEPSEEK_API_KEY`。

## 只部署静态网页

如果部署到 GitHub Pages、普通 OSS、COS、七牛云静态网站等纯静态平台，页面可以打开，但“生成初稿”无法调用 DeepSeek 代理接口。

纯静态部署步骤仍然是：

1. 在本机生成发布文件：

   ```bash
   npm run build
   ```

2. 把生成的 `dist` 文件夹上传到静态网站平台。

3. 得到网址后，可以使用手动排版和导出功能。

手机安装方式：

- iPhone：用 Safari 打开网址，点分享按钮，选择“添加到主屏幕”。
- Android：用 Chrome 打开网址，选择“安装应用”或“添加到主屏幕”。
- 电脑：用 Chrome 或 Edge 打开网址，地址栏右侧通常会出现安装按钮。

## 临时方式：同一 Wi-Fi 内使用

如果只是临时让身边的手机或另一台电脑试用，可以运行：

```bash
npm run dev
```

然后用同一 Wi-Fi 下的设备访问本机局域网地址。这个方式适合临时测试，不适合作为长期使用入口。

## 如果需要真正的安装包

如果希望做成 macOS / Windows 双击安装的软件，可以在这个网页程序外面再套一层桌面壳，比如 Tauri 或 Electron。手机端仍建议保留网页/PWA 方案，因为它最省维护，也最容易分享给别人。
