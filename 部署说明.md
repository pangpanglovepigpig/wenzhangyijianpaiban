# 文章排版工具部署说明

这个项目可以部署成一个网页应用。部署成功后，电脑、手机和平板都能通过同一个网址使用；在支持的浏览器里也可以添加到桌面，像普通 App 一样打开。

当前先推荐部署“无 AI 静态版”：保留文章输入、本地排版、样式设置、预览和下载图片，不显示“生成初稿”。这种方式不需要服务器，也不需要 DeepSeek API Key，更适合先解决国内网络能打开的问题。

如果以后要恢复“生成初稿”，才需要部署带服务器接口的完整 AI 版。完整 AI 版需要 `/api/generate-draft` 代理 DeepSeek，不能只放到普通静态网站。

- EdgeOne Pages Vite 部署说明：https://pages.edgeone.ai/document/vite

- Vite 静态部署说明：https://vite.dev/guide/static-deploy
- GitHub Pages 发布源说明：https://docs.github.com/en/pages/getting-started-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site

## 当前推荐：无 AI 静态版

这个版本适合当前目标：国内用户先能打开网页，并使用排版和导出图片功能。

### 方式 A：EdgeOne Pages

1. 打开 EdgeOne Pages。
2. 选择从 Git 仓库导入项目。
3. 选择仓库：

   ```text
   pangpanglovepigpig/wenzhangyijianpaiban
   ```

4. 框架选择：

   ```text
   Vite
   ```

5. 构建命令填写：

   ```text
   npm run build
   ```

6. 输出目录填写：

   ```text
   dist
   ```

7. 环境变量填写：

   ```text
   VITE_ENABLE_AI_DRAFT=false
   VITE_BASE_PATH=/
   ```

8. 开始部署。
9. 部署完成后，用 EdgeOne Pages 给出的默认域名打开网站。
10. 测试文章输入、点击 `排版文章`、排版编辑、刷新预览、保存全部图片。

静态版不会显示“生成初稿”，也不会调用 `/api/generate-draft`。输入文章后，点击 `排版文章` 会用本地规则生成排版编辑区内容。

### 方式 B：GitHub Pages 备用入口

项目已经准备了 GitHub Actions 发布配置。推送到 `main` 后，GitHub 会构建 `dist` 并发布到 GitHub Pages。

GitHub Pages 构建时会使用：

```text
VITE_ENABLE_AI_DRAFT=false
VITE_BASE_PATH=/wenzhangyijianpaiban/
```

备用地址是：

```text
https://pangpanglovepigpig.github.io/wenzhangyijianpaiban/
```

如果 GitHub Pages 没有自动发布，进入 GitHub 仓库的 `Settings` -> `Pages`，把发布源改成 `GitHub Actions`。

## 二、获取 DeepSeek API Key

1. 打开 DeepSeek 平台。
2. 登录你的 DeepSeek 账号。
3. 进入 API Keys 或密钥管理页面。
4. 点击创建新的 API Key。
5. 复制这串 Key。
6. 先临时放在安全的地方，例如密码管理器或本地备忘录。

注意：

- 不要把 DeepSeek API Key 发给别人。
- 不要把 DeepSeek API Key 写进代码。
- 不要把 DeepSeek API Key 上传到 GitHub。
- 后面只把它填到 Vercel 的环境变量里。

## 七、配置 Vercel 环境变量

在 Vercel 导入项目的配置页面里，找到 `Environment Variables`。

先添加必填变量：

```text
Name:  DEEPSEEK_API_KEY
Value: 你的 DeepSeek API Key
```

然后添加可选变量。建议添加模型变量，方便以后看得清楚：

```text
Name:  DEEPSEEK_MODEL
Value: deepseek-v4-flash
```

说明：

- 不设置 `DEEPSEEK_MODEL` 时，项目默认也是 `deepseek-v4-flash`。
- `deepseek-v4-flash` 生成更快，适合当前“生成初稿”功能。
- 如果更看重质量，且能接受更久等待，可以改成 `deepseek-v4-pro`。

再添加可选超时时间：

```text
Name:  DEEPSEEK_TIMEOUT_MS
Value: 55000
```

说明：

- `55000` 表示等待 55 秒。
- 不设置时默认也是 55 秒。
- 最大建议不要超过 `55000`。
- 如果 DeepSeek 超时、繁忙或返回格式异常，系统会提示失败并保留当前编辑内容，不会再自动生成本地规则初稿。
- 如果线上已经设置过 `DEEPSEEK_TIMEOUT_MS`，请确认它不是 `30000` 这类更短的旧值。

一般不建议开启本地规则兜底。如果确实需要临时应急，可以额外添加：

```text
Name:  DEEPSEEK_ENABLE_LOCAL_FALLBACK
Value: true
```

说明：

- 不设置时默认关闭。
- 开启后，DeepSeek 超时、繁忙或返回格式异常时，系统会用本地规则生成一版可编辑初稿。
- 本地规则只适合作为临时占位，质量不会达到 AI 生成效果。

添加环境变量时，环境范围建议选择：

```text
Production
Preview
Development
```

如果页面只让你选择一种，至少要选 `Production`，这样正式网址才能使用 AI 初稿。

2. 左侧能看到文章输入框。

如果出现错误提示：


## 十一、修改环境变量后必须重新部署

这一点很重要。

如果你在 Vercel 改了这些环境变量：

```text
DEEPSEEK_API_KEY
DEEPSEEK_MODEL
DEEPSEEK_TIMEOUT_MS
DEEPSEEK_ENABLE_LOCAL_FALLBACK
```

旧部署不会自动使用新值。你需要重新部署一次。

重新部署步骤：

1. 打开 Vercel Dashboard。
2. 进入当前项目。
3. 点击左侧或顶部的 `Deployments`。
4. 找到最新一次部署。
5. 点击右侧的三个点。

7. 等待部署完成。
8. 打开线上网址重新测试。

## 十二、查看错误日志

如果页面能打开，但“生成初稿”失败，按下面步骤看日志：

1. 打开 Vercel Dashboard。
2. 进入当前项目。
3. 点击 `Deployments`。
4. 打开最新一次部署。
5. 找到 `Functions` 或 `Runtime Logs`。
6. 找到 `/api/generate-draft` 相关日志。
7. 查看错误内容。

常见错误和处理：

1. 错误提示 `还没有配置 DEEPSEEK_API_KEY。`

   说明 Vercel 没有读到 DeepSeek API Key。去 `Settings` -> `Environment Variables` 检查 `DEEPSEEK_API_KEY`，然后重新部署。

2. 错误提示 `DeepSeek 请求失败：401`

   说明 API Key 不正确、过期，或复制时多了空格。重新复制 DeepSeek API Key，更新 Vercel 环境变量，然后重新部署。

3. 错误提示 `DeepSeek 当前请求过于频繁或额度受限`

   说明请求太频繁或额度受限。稍后再试，或检查 DeepSeek 账户额度。

4. 错误提示 `DeepSeek 当前响应较慢或繁忙`

   说明 DeepSeek 当次响应慢。当前项目默认不会自动生成本地规则初稿，避免低质量内容覆盖编辑区。确认 `DEEPSEEK_TIMEOUT_MS` 是 `55000`，然后重新部署。

5. 浏览器里出现 `/api/generate-draft` 的 `404`

   说明没有部署到支持接口的平台，或者 `api/generate-draft.js` 没有上传到 GitHub。确认使用的是 Vercel，并检查 GitHub 仓库里有没有 `api/` 文件夹。

6. 构建失败并提到 Node 版本

   当前项目要求 Node.js 至少是 `20.19.0`。在 Vercel 项目设置里检查 Node.js 版本，选择较新的 Node 版本后重新部署。
